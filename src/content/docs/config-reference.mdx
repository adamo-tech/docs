Full reference for every field in `config.yaml`. This page is the authoritative source — check here when you need a specific knob.

## Top-Level

```yaml
api_key: "ak_xxxxx"
robot_name: "lab-robot"
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `api_key` | string | — | API key for connecting to Adamo cloud. Fetches the router endpoint and org slug from the API at startup. Required unless `zenoh.router` is set directly. |
| `robot_name` | string | auto-generated | Robot identifier shown in the web interface. Used in topic paths: `{robot_name}/video/{track}`. If omitted, a random name is generated at startup (e.g., `robot-a7k9m`). |

## zenoh

Controls how the robot connects to the network.

```yaml
zenoh:
  mode: "client"
  protocol: "quic"
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `router` | string | — | Direct router endpoint (e.g., `quic/router.adamohq.com:443`). Bypasses the API lookup if set. Useful for self-hosted routers or testing. |
| `mode` | string | `"peer"` | `"client"` routes all traffic through the cloud router. `"peer"` enables direct P2P connections between robots on the same LAN or VPN; the cloud router is still used for discovery and as a fallback when direct paths are unavailable. |
| `listen` | string[] | `[]` | Local endpoints this robot listens on in peer mode, allowing other peers to connect to it directly. Example: `["udp/0.0.0.0:7447", "tcp/0.0.0.0:7447"]`. Has no effect in `"client"` mode. |
| `protocol` | string | `"quic"` | Transport protocol for the cloud router connection: `"quic"`, `"tcp"`, or `"udp"`. QUIC is recommended — it multiplexes streams over a single UDP socket and handles NAT traversal well. |

## control

Controls the inbound control relay. When enabled, control messages published to the configured prefix are delivered to the robot's local handlers.

```yaml
control:
  enabled: true
  zenoh_prefix: "adamo/my-robot/control"
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `true` | Enable the inbound control relay. |
| `zenoh_prefix` | string | `"adamo/{robot}/control"` | Key prefix to subscribe to for incoming control messages. Defaults to `adamo/{robot_name}/control` using the resolved robot name. Override this if you need a custom topic path. |

## video_tracks

An array of video track configurations. Each entry defines one video stream the robot publishes.

```yaml
video_tracks:
  - name: "front"
    source_type: "v4l2"
    v4l2_device: "/dev/video0"
    encoder: "auto"
    bitrate: 4000
    fps: 30
```

### Core Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | string | `"video"` | Track name. Appears in the topic path: `{robot}/video/{name}`. Use short, descriptive names like `"front"`, `"rear"`, `"depth"`. |
| `source_type` | string | `"v4l2"` | Video source — `"v4l2"` (USB / RealSense / any V4L2 device) or `"shm"` (frames produced over an iceoryx2 service). |
| `encoder` | string | `"auto"` | H.264 encoder. Use `"auto"` to let Adamo pick the best available hardware encoder for the host. |
| `bitrate` | int | `2000` | Target bitrate in kbps. Starting bitrate for adaptive bitrate tracks. |
| `fps` | int | `30` | Target framerate. |

### Source-Specific Fields

| Field | Applies to | Type | Description |
|-------|-----------|------|-------------|
| `v4l2_device` | `v4l2` | string | Device path (e.g., `"/dev/video0"`). |
| `v4l2_capture_resolution` | `v4l2` | [int, int] | Capture resolution as `[width, height]`. Must match a resolution the camera supports. |
| `shm_service` | `shm` | string | iceoryx2 service name to subscribe to. |
| `source_format` | both | string | Pixel format hint (e.g., `"BGRA"`, `"YUY2"`, `"NV12"`). On Jetson, V4L2 sources auto-detect. |

### Encoder Selection

| Host                            | Encoder used |
|---------------------------------|--------------|
| Jetson (Orin, Thor, …)          | On-chip NVENC |
| x86 with NVIDIA discrete GPU    | NVENC |
| Intel / AMD integrated graphics | VA-API |
| macOS                           | VideoToolbox |
| Anything else                   | CPU (x264) |

`encoder: "auto"` (the default) picks the best available. To pin a specific encoder, set `encoder` to the appropriate element name.

### Advanced Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `keyframe_distance` | float | `-1.0` | Seconds between periodic keyframes. `-1.0` disables periodic keyframes entirely — keyframes are only sent on-demand in response to PLI or FIR requests from the receiver. Set to a positive value (e.g., `2.0`) if the stream must be joinable mid-stream without a request. |
| `stereo` | bool | `false` | Mark this track as a top/bottom stereo stream for VR viewing. |
| `fec` | bool | `false` | Enable Forward Error Correction. Adds parity packets so receivers can reconstruct lost packets without retransmission. Increases bandwidth by `fec_parity_shards / fec_data_shards` (50% with defaults). |
| `fec_data_shards` | int | `4` | Number of data packets per FEC group. |
| `fec_parity_shards` | int | `2` | Number of parity packets per FEC group. With 4 data + 2 parity, the stream survives any 2 lost packets per group. |
| `nack` | bool | `false` | Enable NACK-based retransmission. The receiver requests re-send of lost packets. Adds latency proportional to RTT. Useful on reliable but occasionally lossy links. |

### `adaptive_bitrate`

Nested under a video track. When enabled, the encoder bitrate adjusts dynamically based on measured network capacity using Google Congestion Control.

```yaml
video_tracks:
  - name: "front"
    source_type: "v4l2"
    v4l2_device: "/dev/video0"
    encoder: "auto"
    bitrate: 4000
    adaptive_bitrate:
      enabled: true
      min_bitrate: 500
      max_bitrate: 8000
      priority: 1.5
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `false` | Enable adaptive bitrate control for this track. |
| `min_bitrate` | int | — | Minimum bitrate in kbps. The encoder will not go below this value even under severe congestion. |
| `max_bitrate` | int | — | Maximum bitrate in kbps. The encoder will not exceed this value even when bandwidth is available. |
| `priority` | float | `1.0` | Bandwidth allocation weight relative to other tracks. A track with `priority: 2.0` gets roughly twice the bandwidth of a `priority: 1.0` track when both are contending for capacity. Higher-priority tracks are also protected first when the network degrades. |

## Full Example

A realistic multi-track configuration with control relay, adaptive bitrate, and FEC:

```yaml
api_key: "ak_xxxxx"
robot_name: "lab-robot"

zenoh:
  protocol: "quic"

control:
  enabled: true

video_tracks:
  - name: "front"
    source_type: "v4l2"
    v4l2_device: "/dev/video0"
    encoder: "auto"
    bitrate: 4000
    fps: 30
    fec: true
    keyframe_distance: -1.0
    adaptive_bitrate:
      enabled: true
      min_bitrate: 500
      max_bitrate: 8000
      priority: 1.5

  - name: "rear"
    source_type: "v4l2"
    v4l2_device: "/dev/video2"
    encoder: "auto"
    bitrate: 2000
    fps: 15
    adaptive_bitrate:
      enabled: true
      min_bitrate: 300
      max_bitrate: 4000
      priority: 1.0
```

### Peer-to-Peer Mode

Robots on the same LAN communicate directly. The cloud router handles discovery and browser connections, but robot-to-robot traffic never leaves the local network:

```yaml
api_key: "ak_xxxxx"
robot_name: "field-robot"

zenoh:
  mode: "peer"
  listen:
    - "udp/0.0.0.0:7447"
    - "tcp/0.0.0.0:7447"
```

### Stereo Camera over Shared Memory

When a stereo camera driver publishes side-by-side frames to an iceoryx2 service, attach it as an `shm` source and mark the track `stereo: true` so viewers know to render it as a VR-friendly top/bottom stream.

```yaml
api_key: "ak_xxxxx"
robot_name: "orin"

zenoh:
  mode: "client"

control:
  enabled: true

video_tracks:
  - name: "main"
    source_type: "shm"
    shm_service: "stereo_cam"
    encoder: "auto"
    bitrate: 8000
    fps: 60
    source_format: "BGRA"
    keyframe_distance: -1.0
    stereo: true
    adaptive_bitrate:
      enabled: true
      min_bitrate: 1000
      max_bitrate: 12000
      priority: 1.0
```
